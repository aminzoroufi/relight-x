# Optional ROS2/Gazebo Skeleton

Unity is the main ReLight-X digital twin. This folder is only a lightweight placeholder for future ROS2/Gazebo experiments.

Possible future uses:

- Publish vehicle poses from a ROS2 traffic simulator.
- Subscribe to backend luminaire brightness topics through a bridge.
- Visualize pole positions and zones in Gazebo.
- Test robotics-style sensor fusion without changing the main Python backend.

This skeleton is intentionally minimal to avoid overcomplicating the prototype.
